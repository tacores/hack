# SSRF

拒否リストを回避する方法

```text
（拒否リストにローカルホストが載っている場合）
0
0.0.0.0
0000
127.1
127.*.*.*
2130706433
017700000001
などの代替ローカルホスト参照

または
127.0.0.1.nip.io
などの 127.0.0.1に解決されるDNSレコードを使って回避可能
```

許可リストを回避する方法

```text
https://website.thm
のみ許可するルールがある場合、自分のドメインにサブドメインを作成することで回避可能。IPアドレス形式のサブドメインも可能。
https://website.thm.attackers-domain.thm
```
